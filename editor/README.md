# シナリオエディタ (editor/)

ゲーム本編とは独立した、シナリオ編集用のローカルツールです。
キー参照になっている台詞テキストを、流れの中で **JA / EN を並列編集** できます。

## 起動方法

### Windows
`editor/start.bat` をダブルクリック。Pythonがインストールされていれば自動で
- `http://127.0.0.1:8765/editor/` をブラウザで開く
- 編集サーバ（プロジェクトルートを配信＋保存API）が起動

### 直接コマンド
```
python editor/server.py
```
ブラウザで `http://127.0.0.1:8765/editor/` を開いてください。

> **要件**: Python 3.8+（標準ライブラリのみ使用、追加インストール不要）

## 編集できること

### Phase 1
- ✅ 全シナリオファイル（`scenarios/*.md`）のラベル一覧表示
- ✅ ラベル選択 → そのラベル内の全行を表示
- ✅ ナレーション / セリフ / 選択肢の **JA / EN 同時編集**
- ✅ 編集内容を `locales/scenario.ja.json` / `locales/scenario.en.json` に保存

### Phase 2
- ✅ 制御コマンドのフォーム編集（`@scene` `@bgm` `@se` `@show` `@hide` `@hide_all` `@expr` `@move` `@still` `@still_hide` `@wait` `@window_color` `@jump` `@flag` `@effect` `@shake`）
  - BG / BGM / SE / Still / キャラ / 表情 はサーバが走査した実ファイル一覧から選択肢化
  - 未対応コマンドは raw テキスト編集にフォールバック
- ✅ コマンド行の編集を `scenarios/*.md` に書き戻し（パース→シリアライズ往復はバイト一致）
- ✅ **静止画プレビュー**: 選択行までを走査して BG + 立ち絵 + セリフ枠を合成表示
  - JA / EN 切替トグル
  - BGM / SE / Window色は右下にテキスト表示
- 未対応: `@if` `@route_select` のフォーム編集 → raw 編集で代替

### Phase 3
- ✅ **行の挿入**: 行間にホバーすると `+` ボタンが出現 → クリックでメニュー
  - ナレーション / セリフ / コマンド / 選択肢ブロック / コメント / 空行
- ✅ **行の削除**: 各行の右上に `✕` ボタン
- ✅ **キー自動採番**:
  - テキスト行: `{file}.{label}.t{NNN}` 形式で次の番号を採番
  - 選択肢: `choices.{file}.{label}_{NN}` 形式
  - 採番と同時に `locales.ja` / `locales.en` に空エントリを追加
- ✅ **セリフ行のキャラ変更**: 話者タグがプルダウンに（player / sakura / 衣装バリエ等）
- ✅ **選択肢ブロック編集**:
  - 分岐先プルダウン（全 .md から収集したラベル一覧）
  - フラグ修飾子のテキスト入力（`sakura_favor+5,kotoha_favor-2` 等）
  - `+ 選択肢を追加` で同ブロックに新規分岐を挿入
- ✅ **ラベル追加**: ファイル名の `+` ボタン → 名前入力で末尾に追加
- ✅ **ラベル削除**: ラベル名の `✕` ボタン（ラベル行のみ削除、配下の行は残る）
- 未対応: 新規 `.md` ファイル作成（`config.js` 編集が必要なため別途）

## 安全策

- サーバは `127.0.0.1` のみバインド（外部からアクセス不可）
- 書込先は `scenarios/*.md` と `locales/*.json` に限定
- パストラバーサル禁止
- 一時ファイル → rename の原子的書込（途中で落ちても元ファイルが残る）
- 保存前にJSONをサーバ側でも構文検証

## ファイル構成

```
editor/
├── README.md            # このファイル
├── start.bat            # 起動スクリプト (Windows)
├── server.py            # 編集サーバ（静的配信 + 保存API）
├── index.html           # エディタUIエントリポイント
├── css/
│   └── editor.css       # スタイル
└── js/
    ├── app.js           # メイン制御
    ├── tree.js          # 左ペイン（ファイル/ラベルツリー）
    ├── line-editor.js   # 中央ペイン（行リスト・テキスト編集）
    ├── command-editor.js# コマンド行のフォーム編集
    ├── preview.js       # 右ペイン（静止画プレビュー）
    ├── scenario-io.js   # .md パース/シリアライズ
    └── api.js           # サーバAPIクライアント
```

ゲーム本体のファイル（`config.js` / `game.js` / `parser.js` 等）は
一切変更しません。`config.js` の `backgrounds` 定義のみ、プレビューの
グラデーションフォールバックに使うため読み取り専用で読み込みます。
