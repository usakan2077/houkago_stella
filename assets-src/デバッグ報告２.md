## システム・UI

**【重大】選択肢表示中にセーブ＆ロードできない**
✅ 対応済み (P0) — `_openSaveLoad()` のガードを削除。ロード時に `@choice` が再実行され選択肢が自動復元される。

**【重大】ゲームをしばらく放置した後でチャプターを終えると次チャプターに入れず END 画面になる**
✅ 対応済み (P0) — `@wait` の `setTimeout` を `_waitTimer` で追跡し、ラベル切り替え時に確実にキャンセルするよう修正。旧実装では古い `@wait` タイマーが新ラベル上で `_executeNext` を発火して END を引き起こしていた。

**イベント画像表示中にセーブ→ロードしてもイベント絵が表示されず前後のシーン画像のまま進む**
✅ 対応済み — `_showStill()` で `effect='instant'` 時に CSS クラスを付与しないよう修正（`effect-instant` に対応するスタイルが未定義だったため表示崩れ）。

**ロード時、画像しか表示されないままスキップかオートしかできない状態になることがある（右クリック UI 非表示起因）**
✅ 対応済み — `_resetGameState()` で `_uiHidden` フラグと `text-area`/`menu-bar` の `visibility` を明示的にリセット。右クリック UI 非表示状態のままセーブ→ロードすると発生していた。

**END に入った後ロードしてニューゲームしても画像しか表示されず操作不能になる**
✅ 対応済み — `_displayEnding()` が `text-area`・`menu-bar` に設定する `opacity:0`/`pointerEvents:none` のインラインスタイルが `_resetGameState()` でクリアされていなかった。`visibility` のリセットだけでは不十分だったため、`opacity`・`transition`・`pointerEvents` も明示的に空文字でリセットするよう修正。

**オート解除のタイミング次第でテキストが一切進まなくなる**
✅ 対応済み — `_toggleAuto()` でオート無効化時に `_stopTypewriter()` を呼んでいたためタイプライターごと停止していた。オート無効化は `autoTimer` のみキャンセルするよう変更。

**イベント絵表示時、テキストウィンドウをクリックしないとテキストが進まない**
✅ 部分対応 — ロード復元時のスチルロック（1.5秒）を除去。通常表示時はロック継続（誤クリック防止のため）。

**バックログで右クリックしてもログウィンドウが閉じない・右クリックメニューが出る。セーブ&ロード画面でも同様**
✅ 対応済み — モーダル全体に `contextmenu` リスナーを追加し、右クリックでモーダルを閉じるよう変更。

**各コマンドボタンに対応キーの表示がない（SAVE: F5、LOAD: F7 など）**
✅ 対応済み — 各ボタンにキー表示を追加。

**テキスト表示が常に1行単位で、しゃべり方が不自然に感じる**
📌 保留 — 仕様として継続検討。

**さくらルート以降の `（）` 付きテキストを表示途中でクリックすると文字サイズや括弧の表示が崩れる**
✅ 対応済み — `currentEmphasis` を追跡し、`_onAdvance()` の即時表示時に `inner` なら `<em>` タグを `innerHTML` で維持するよう修正。

**テキストが瞬間的に2回表示されることがある（フォント警告も同時に出る）**
✅ 対応済み — `_displayDialog()` 冒頭で `autoTimer` をキャンセルするよう修正。オートモード中にクリックで手動進行したとき、前行の `autoTimer` が残存して新ダイアログの直後に `_executeNext()` を再度呼び次行テキストを瞬間表示してしまっていた。フォント警告は別原因（WebフォントFOUT）の可能性があり並行して発生していたと考えられる。

**BGM が重複して再生される**
✅ 対応済み — 2つの原因を修正：
- `_playBGM()` の autoplay ブロック時 `catch` 内で `resume` コールバックを `document` に複数登録していた。`_playBGM()` が複数回呼ばれると古い Audio の resume も残存し、最初のクリックで全て一斉再生されていた → `resume` 内に `if (this.bgmAudio !== audio) return;` ガードを追加。
- `_startTitleBGM()` が `this.bgmAudio` を直接置き換えており、fade-out 処理を経由しない独立した管理になっていた → `_playBGM()` に統合し BGM 管理を一元化。

**キャラクターごとのウィンドウカラー設定がない**
✅ 対応済み — `_setWindowColor()` メソッドを実装。各ルート（さくら・ことは・まひる）で対応色に自動切り替わる機能を追加。`@window_color` コマンドで手動制御も可能。ロード時に選択色を復元。

**その他修正まとめ**
- `assets-src/chars/sakura/sport/` の古い画像ファイルをクリーンアップ（`OLD/` フォルダと旧表情ファイルを削除）
- 新規 BGM・SE 追加（`piano_stage`、`applause` 等）
- 背景画像追加（`school_grounds_evening`）
- `assets-src/bg_re` の背景31枚を `assets/images/bg` 側へ反映（元 PNG を 1920x1080 にリサイズして WebP 化し、ファイル名揺れも整理）
