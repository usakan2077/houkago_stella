
システム・UI

情報：スマホブラウザプレイ時・スキップやオート解除後、他の場所を一度タップするまでコマンドが選択状態(オンマウス状態(ピンクの枠が出てる状態))なので解除されたかどうかわかりにくい。
→ 対応方針：
  ・CSS: `@media (hover: none)` で `.menu-btn:hover` スタイルを無効化
    （タッチデバイスではホバー状態が貼り付くため）
  ・JS: `_toggleSkip()` / `_toggleAuto()` の末尾に `document.activeElement.blur()`
    を追加してフォーカスを外す
→ 対応内容：
  ・`style.css` に `@media (hover: none)` を追加し、`menu-btn` の hover 表現がスマホで残留しないよう調整
  ・`game.js` の `_toggleSkip()` / `_toggleAuto()` の末尾で `document.activeElement.blur()` を実行するよう修正

情報：PCブラウザと違いカーソルが追従しない。タップしたときだけそこに動く感じ。
  →スマホブラウザだとカーソル邪魔な気がしてきた。
→ 対応方針：
  ・カーソル非表示
    `@media (hover: none) and (pointer: coarse)` でタッチデバイスを検出し、
    ゲーム画面内の全要素に `cursor: none` を適用。
  ・タップ時に星エフェクトを追加（カーソルの代替フィードバック）
    `touchstart` イベントでタップ座標に星のパーティクルを生成。
    星は数個（5〜8個）がランダム方向に飛び散って消えるアニメーション。
    要素は `<span>` を動的生成して `#game-screen` に追加、
    アニメーション終了後に自動削除（`animationend` で removeChild）。
    PC のカスタムカーソルエフェクトと別系統で管理し、
    `hover: none` 環境でのみ動作するよう JS 側でタッチ検出して制御。
→ 対応内容：
  ・PC 用カスタムカーソルは `hover: hover` かつ `pointer: fine` 前提に整理し、タッチ端末では `_initCursor()` をスキップ
  ・スマホ向けには `#game-screen` 配下のみ `cursor: none` を適用
  ・`touchstart` 時に `#game-screen` 上へ `touch-star` を 5〜8 個生成し、星が散って消える演出を追加

情報：右クリックがないからテキストウィンドウオフができない。
→ 対応方針：
  ・長押し（300ms）で右クリックと同じ UI 非表示トグルを発火
  ・`touchstart` でタイマーセット、`touchend` / `touchmove` でキャンセル
    （`touchmove` でキャンセルすることでスクロールとの誤検知を防ぐ）
  ・JS 実装は約 20 行、既存の `contextmenu` 処理と同じ `_hideUI()` / `_showUI()` を呼ぶだけ
→ 対応内容：
  ・`game.js` の `#game-screen` に対して `touchstart` / `touchend` / `touchcancel` / `touchmove` を追加
  ・300ms の長押しで `_hideUI()` / `_showUI()` を切り替える実装を追加
  ・長押し直後の click で意図せず UI が戻らないよう、長押し発火後は一時的に click を無視するガードを追加

情報：俺の端末のスペック不足なのかもしれないが結構重め+熱を持つ。
→ 対応方針：
  ・`@media (hover: none) and (pointer: coarse)` でスマホ向けに以下を適用
    - タイトル画面の infinite アニメーション（titlePulse / starTwinkle / titleGlow / beamPulse）を無効化または間引く
    - ゲーム中の arrowBounce（↓マーク）を duration を長くするか opacity 点滅に変更して GPU 負荷を下げる
    - `will-change: transform` のついた要素を `will-change: auto` に戻す
    - box-shadow の多重掛けを簡略化
  ・`prefers-reduced-motion` にも対応できれば同時対応
→ 対応内容：
  ・スマホ向けメディアクエリで `titlePulse` / `starTwinkle` / `titleGlow` / `beamPulse` を停止
  ・`#next-arrow` は `arrowBounce` から軽めの opacity 点滅アニメーションへ変更し、`#opening-arrow` も周期を長めに調整
  ・`#scene-wipe` / `#credits-roll` の `will-change` をスマホ・ reduced motion 環境で `auto` に戻すよう調整
  ・タイトル周辺と `#game-container` の影表現を少し簡略化し、`prefers-reduced-motion: reduce` でも主要アニメーションを止める対応を追加
