# スマホ版デバッグ報告

---

## システム・UI

**スキップ/オート解除後、ボタンが選択状態（ピンク枠）のまま残る**
✅ 対応済み —
- CSS: `@media (hover: none)` で `.menu-btn:hover` スタイルを無効化（タッチデバイスではホバー状態が貼り付くため）
- JS: `_toggleSkip()` / `_toggleAuto()` の末尾に `document.activeElement.blur()` を追加してフォーカスを外す

**カーソルがタップ位置に残って邪魔**
✅ 対応済み —
- **カーソル非表示**: `@media (hover: none) and (pointer: coarse)` でタッチデバイスを検出し、ゲーム画面内の全要素に `cursor: none` を適用
- **タップ時に星エフェクトを追加**（カーソルの代替フィードバック）: `touchstart` イベントでタップ座標に星のパーティクルを生成。星は5〜8個がランダム方向に飛び散って消えるアニメーション。要素は `<span>` を動的生成して `#game-screen` に追加し、`animationend` で自動削除。`hover: none` 環境でのみ動作するよう JS 側でタッチ検出して制御
- **バグ修正**: `prefers-reduced-motion: reduce` 時にアニメーションが無効化されて `animationend` が発火せず星が残留する問題を修正。`prefers-reduced-motion` が有効な場合は `_spawnTouchParticles()` を early return するよう変更。フォールバック `setTimeout(900ms)` も追加し、`animationend` が発火しない場合でも確実に削除されるよう対応。

**右クリックがないためテキストウィンドウオフができない**
✅ 対応済み —
- 長押し（300ms）で右クリックと同じ UI 非表示トグルを発火
- `touchstart` でタイマーセット、`touchend` / `touchmove` でキャンセル（`touchmove` でキャンセルすることでスクロールとの誤検知を防ぐ）
- 既存の `contextmenu` 処理と同じ `_hideUI()` / `_showUI()` を呼ぶ実装

**重め・発熱する（端末スペックにもよる）**
✅ 対応済み —
- `@media (hover: none) and (pointer: coarse)` でスマホ向けに以下を適用:
  - タイトル画面の infinite アニメーション（`titlePulse` / `starTwinkle` / `titleGlow` / `beamPulse`）を無効化または間引く
  - ゲーム中の `arrowBounce`（↓マーク）を `arrowFadePulse`（opacity 点滅）に変更して GPU 負荷を下げる
  - `will-change: transform` のついた要素を `will-change: auto` に戻す
  - `box-shadow` の多重掛けを簡略化
- `prefers-reduced-motion: reduce` にも対応済み（アニメーション全般を無効化）
